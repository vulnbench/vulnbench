Difficulty Tier & Ecosystem Breakdown

Per-tier and per-ecosystem breakdowns are not available for best-of-3 aggregate results. The benchmark classifies instances into three difficulty tiers (Tier 1: Pattern, Tier 2: Logic, Tier 3: Deep) with 67/67/66 instances each, and spans 7 ecosystems (npm, pip, Maven, RubyGems, Composer, Rust, Swift). See individual single-run result files for tier-level and ecosystem-level analysis.

Methodology

Dataset

200 real CVE instances from the GitHub Advisory Database, covering 200 unique repos across 7 ecosystems (npm, pip, Maven, RubyGems, Composer, Rust, Swift) and 55 CWE types. Balanced across 3 difficulty tiers. Mean patch size: 36 lines across 1.9 files.

Prompt

Each model receives CVE text, CWE guidance, and runtime source context. File hints are derived from advisory text only (description mode). Each model is run 3 times per instance (best-of-3) for variance reduction.

Scoring (LLM-as-Judge)

Claude Opus 4.6 compares each candidate patch against the ground-truth fix. Scores 0.0-1.0 based on: root cause addressed, no new vulnerabilities introduced, scope coverage. Pass threshold: score ≥ 0.5 or judge verdict "pass".
